# Claude Code Configuration

## Project

Learning project to explore backend architecture patterns (WebSockets, Pub/Sub, Message Queues, Events) using a fitness app as the domain. Stack: NestJS, Socket.io, React, MongoDB (planned).


## Documentation

- When adding a new feature or architecture pattern, update `docs/architecture.md` to reflect it
- Keep the module table, event tables, and roadmap in `docs/architecture.md` current


## Working Conventions
- After completing any feature change, update the relevant doc in docs/features/
- After adding a new feature, create docs/features/<feature-name>.md
- Keep docs navigational — file paths, data flow, gotchas. Not full API docs.
